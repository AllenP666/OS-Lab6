# Лабораторная работа 6: Графический клиент (GUI)

Данный проект является продолжением 5-й лабораторной работы. К существующему HTTP-серверу с базой данных SQLite добавлено кроссплатформенное десктопное приложение с графическим интерфейсом (GUI) на C++ с использованием библиотеки **Qt (Qt Charts)**.

## Архитектура
1. **Симулятор датчика:** Генерирует случайные значения температуры и пишет их в виртуальный COM-порт (файл).
2. **Сервер (Backend):** Читает данные, сохраняет в SQLite и предоставляет REST API (`/api/current`, `/api/history`).
3. **GUI Клиент (Frontend):** Desktop-приложение на Qt. Запрашивает данные у сервера по HTTP и отрисовывает графики в реальном времени.

## Требования
Для сборки необходимы **CMake**, компилятор **C++17** и библиотека **Qt6** (компоненты: Widgets, Network, Charts).

### Установка зависимостей
- **Windows (MSYS2 MinGW64):**
  ```bash
  pacman -S mingw-w64-x86_64-cmake mingw-w64-x86_64-toolchain mingw-w64-x86_64-qt6-base mingw-w64-x86_64-qt6-charts
  ```
- **Linux (Ubuntu 20.04+):**
  ```bash
  sudo apt update
  sudo apt install build-essential cmake qtbase5-dev libqt5charts5-dev
  ```

## Сборка проекта

### Windows (через MSYS2)
Откройте терминал **MSYS2 MinGW x64** и выполните:
```bash
sh build.sh
```

### Linux
Откройте терминал в папке проекта и выполните:
```bash
sh build.sh
```

## Запуск

Используйте автоматический скрипт, который запускает симулятор, сервер и GUI-клиент.

- **Windows (в терминале MSYS2):** `sh run_all.sh`
- **Linux:** `sh run_all.sh`

## Структура проекта
- `gui_client/` — исходный код GUI приложения (Qt).
- `main.cpp`, `http_server.hpp` — исходный код сервера.
- `database.hpp` — работа с SQLite (хранение логов).
- `simulator.cpp` — эмулятор датчика.
- `build.sh` — универсальный скрипт сборки.
- `run_all.sh` — скрипт запуска всех компонентов.
